41/**
42 * Serializes Rule objects back to LDAP Filter DSL expression strings.
43 *
44 * Provides reverse transformation from compiled Rule/Proposition trees back
45 * to human-readable LDAP filter syntax following RFC 4515. Supports all LDAP
46 * operators including comparison (=, >=, <=, ~=), logical (&, |, !), and
47 * special filters (presence, wildcard).
48 *
49 * Pattern: Each DSL should provide three public classes:
50 * - {DSL}Parser: Parse DSL strings → Rule objects
51 * - {DSL}Serializer: Serialize Rule objects → DSL strings
52 * - {DSL}Validator: Validate DSL strings without full parsing
53 *
54 * LDAP uses prefix notation where operators precede their operands:
55 * - AND: (&(age>=18)(country=US))
56 * - OR: (|(role=admin)(role=manager))
57 * - NOT: (!(status=inactive))
58 * - Presence: (email=*)
59 * - Wildcard: (name=John*)
60 *
61 * Example usage:
62 * ```php
63 * $serializer = new LDAPFilterSerializer();
64 * $parser = new LDAPFilterParser();
65 *
66 * $rule = $parser->parse('(&(age>=18)(country=US))');
67 * $expression = $serializer->serialize($rule);
68 * // Returns: '(&(age>=18)(country=US))'
69 * ```
70 *
71 * @author Brian Faust <brian@cline.sh>
72 *
73 * @see LDAPFilterParser For parsing LDAP filter strings into Rules
74 * @see LDAPFilterValidator For validating LDAP filter strings
75 *
76 * @psalm-immutable
77 */
