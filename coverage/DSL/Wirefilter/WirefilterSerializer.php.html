57/**
58 * Serializes Rule objects back to Wirefilter DSL expression strings.
59 *
60 * Provides reverse transformation from compiled Rule/Proposition trees back
61 * to human-readable Wirefilter DSL syntax. Supports all Wirefilter operators
62 * including comparison, logical, mathematical, and string operations.
63 *
64 * Pattern: Each DSL should provide three public classes:
65 * - {DSL}Parser: Parse DSL strings → Rule objects
66 * - {DSL}Serializer: Serialize Rule objects → DSL strings
67 * - {DSL}Validator: Validate DSL strings without full parsing
68 *
69 * Example usage:
70 * ```php
71 * $serializer = new WirefilterSerializer();
72 * $parser = new WirefilterParser();
73 *
74 * $rule = $parser->parse('age >= 18 && country == "US"');
75 * $expression = $serializer->serialize($rule);
76 * // Returns: 'age >= 18 && country == "US"'
77 * ```
78 *
79 * @author Brian Faust <brian@cline.sh>
80 *
81 * @see WirefilterParser For parsing DSL strings into Rules
82 * @see WirefilterValidator For validating DSL strings
83 *
84 * @psalm-immutable
85 */
